# Grok 视频接口规范

## 1. 基本信息

| 项目 | 值 |
| :-- | :-- |
| Base URL | `https://<your-host>` |
| 认证方式 | HTTP Header `Authorization: Bearer <api_key>` |
| Content-Type | `application/json`（视频接口额外支持 `multipart/form-data`） |
| 视频模型 | `grok-imagine-video` |

---

## 2. POST /v1/videos — 视频生成

异步提交视频生成任务，立即返回任务 ID 和初始状态。同时支持 `application/json` 和 `multipart/form-data` 两种请求格式。

### 2.1 请求字段

| 字段 | 类型 | 必填 | 取值范围 | 说明 |
| :-- | :-- | :-- | :-- | :-- |
| `model` | string | 是 | `grok-imagine-video` | 固定值 |
| `prompt` | string | 是 | - | 视频描述文本 |
| `seconds` | int/string | 否 | `6` / `10` / `12` / `16` / `20` | 视频时长（秒），默认 6 |
| `size` | string | 否 | `720x1280` / `1280x720` / `1024x1024` / `1024x1792` / `1792x1024` | 视频尺寸，默认 `720x1280` |
| `resolution_name` | string | 否 | `480p` / `720p` | 分辨率名称 |
| `preset` | string | 否 | `fun` / `normal` / `spicy` / `custom` | 风格预设 |
| `image_reference` | object | 否 | `{ "image_url": "<URL 或 data URI>" }` | **JSON 格式 · 单图**图生视频参考图 |
| `image_references` | array | 否 | `[ { "image_url": "..." }, ... ]` | **JSON 格式 · 多图**图生视频参考图数组 |
| `input_reference` | file | 否 | 单个文件 | **Multipart 格式 · 单图**图生视频参考图 |
| `input_references` | file[] | 否 | 多个文件（重复字段名） | **Multipart 格式 · 多图**图生视频参考图 |

> **注意**：单图字段（`image_reference` / `input_reference`）与多图字段（`image_references` / `input_references`）不可混用。JSON 和 Multipart 格式各自使用对应字段名。

### 2.2 JSON 格式示例

#### 文生视频

```json
{
  "model": "grok-imagine-video",
  "prompt": "霓虹雨夜街头，电影感慢镜头追拍",
  "seconds": 10,
  "size": "1792x1024",
  "resolution_name": "720p",
  "preset": "normal"
}
```

#### 单图图生视频

```json
{
  "model": "grok-imagine-video",
  "prompt": "让这张照片里的人物转身微笑",
  "seconds": 6,
  "image_reference": {
    "image_url": "https://example.com/ref.jpg"
  }
}
```

#### 多图图生视频

```json
{
  "model": "grok-imagine-video",
  "prompt": "从第一张镜头平滑过渡到第二张",
  "seconds": 10,
  "image_references": [
    { "image_url": "https://example.com/a.jpg" },
    { "image_url": "data:image/png;base64,iVBORw0K..." }
  ]
}
```

### 2.3 Multipart 格式示例

#### 单图图生视频

```bash
curl https://<your-host>/v1/videos \
  -H "Authorization: Bearer $API_KEY" \
  -F "model=grok-imagine-video" \
  -F "prompt=参考这张照片生成视频" \
  -F "seconds=6" \
  -F "input_reference=@/path/to/ref.jpg"
```

#### 多图图生视频

```bash
curl https://<your-host>/v1/videos \
  -H "Authorization: Bearer $API_KEY" \
  -F "model=grok-imagine-video" \
  -F "prompt=从 A 过渡到 B" \
  -F "seconds=10" \
  -F "input_references=@/path/a.jpg" \
  -F "input_references=@/path/b.jpg"
```

### 2.4 响应字段

| 字段 | 类型 | 说明 |
| :-- | :-- | :-- |
| `id` | string | 视频任务 ID（格式如 `vg_abc123...`） |
| `object` | string | 固定值 `video.generation` |
| `created_at` | int64 | 创建时间（Unix 时间戳） |
| `status` | string | 任务状态：`queued` / `processing` / `completed` / `failed` |
| `model` | string | 使用的模型名 |
| `progress` | int | 进度百分比（0-100） |
| `prompt` | string | 输入的视频描述 |
| `seconds` | string | 视频时长 |
| `size` | string | 视频尺寸 |
| `quality` | string | 分辨率（如 `720p`） |

### 2.5 响应示例

```json
{
  "id": "vg_abc123def456",
  "object": "video.generation",
  "created_at": 1714780800,
  "status": "queued",
  "model": "grok-imagine-video",
  "progress": 0,
  "prompt": "霓虹雨夜街头，电影感慢镜头追拍",
  "seconds": "10",
  "size": "1792x1024",
  "quality": "720p"
}
```

## 3. GET /v1/videos/{video_id} — 查询任务状态

轮询视频生成任务的当前状态。建议轮询间隔 2-5 秒，当状态为 `completed` 或 `failed` 时停止轮询。

### 3.1 请求说明

```
GET /v1/videos/{video_id}
Authorization: Bearer <api_key>
```

- `video_id`：创建任务时返回的 `id` 字段值

### 3.2 响应字段

| 字段 | 类型 | 说明 |
| :-- | :-- | :-- |
| `id` | string | 视频任务 ID |
| `object` | string | `video.generation` |
| `status` | string | 当前状态 |
| `progress` | int | 进度百分比（0-100） |
| `created_at` | int64 | 创建时间 |
| `completed_at` | int64 | 完成时间（仅 `completed` 状态） |
| `video_url` | string | 视频下载地址（仅 `completed` 状态） |
| `error` | object | 错误信息（仅 `failed` 状态） |
| `error.code` | string | 错误代码 |
| `error.message` | string | 错误描述 |

### 3.3 各状态响应示例

#### queued（已入队）

```json
{
  "id": "vg_abc123def456",
  "object": "video.generation",
  "status": "queued",
  "progress": 0,
  "created_at": 1714780800,
  "model": "grok-imagine-video"
}
```

#### processing（生成中）

```json
{
  "id": "vg_abc123def456",
  "object": "video.generation",
  "status": "processing",
  "progress": 45,
  "created_at": 1714780800,
  "model": "grok-imagine-video"
}
```

#### completed（已完成）

```json
{
  "id": "vg_abc123def456",
  "object": "video.generation",
  "status": "completed",
  "progress": 100,
  "created_at": 1714780800,
  "completed_at": 1714781400,
  "model": "grok-imagine-video",
  "video_url": "https://cdn.example.com/videos/vg_abc123def456.mp4"
}
```

#### failed（已失败）

```json
{
  "id": "vg_abc123def456",
  "object": "video.generation",
  "status": "failed",
  "progress": 0,
  "created_at": 1714780800,
  "model": "grok-imagine-video",
  "error": {
    "code": "upstream_failure",
    "message": "Video generation failed due to content policy violation"
  }
}
```

## 4. GET /v1/videos/{video_id}/content — 下载视频

### 4.1 请求说明

```
GET /v1/videos/{video_id}/content
Authorization: Bearer <api_key>
```

- 仅在任务状态为 `completed` 时调用
- 支持 session auth（后台面板）和 token auth（API 客户端）

### 4.2 响应说明

- **Content-Type**: `video/mp4`
- **响应体**: MP4 视频文件二进制流
- 建议使用 `-L` 跟随重定向，`-o` 指定输出文件

```bash
curl -L https://<your-host>/v1/videos/<video_id>/content \
  -H "Authorization: Bearer $API_KEY" \
  -o result.mp4
```

## 5. POST /v1/videos/{video_id}/remix — Remix

基于已有视频生成新的变体视频。

### 5.1 请求字段

| 字段 | 类型 | 必填 | 说明 |
| :-- | :-- | :-- | :-- |
| `prompt` | string | 是 | 新的视频描述 |

### 5.2 请求示例

```bash
curl https://<your-host>/v1/videos/<video_id>/remix \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "将视频风格改为赛博朋克风"
  }'
```

### 5.3 响应

返回格式与 `POST /v1/videos` 相同，包含新任务的 `id` 和初始状态。响应中会包含 `remixed_from_video_id` 字段指向原视频 ID。

## 6. 计费规则

### 6.1 文生视频 / 单图图生视频积分表

| 时长 | 积分 |
| :-- | :-- |
| 6s | 60 |
| 10s | 100 |
| 12s | 140 |
| 16s | 240 |
| 20s | 500 |

### 6.2 多图图生视频积分表（4/3 倍率）

上传 **≥2 张**参考图时，在单图/文生价格基础上 × 4/3，向下取整。

| 时长 | 积分 |
| :-- | :-- |
| 6s | 80 |
| 10s | 133 |
| 12s | 186 |
| 16s | 320 |
| 20s | 666 |

### 6.3 计费说明

- 单图图生视频与文生视频计费**完全一致**，不加价
- 仅上传 **≥2 张**参考图时触发 4/3 倍率
- 积分在请求入口处**预扣**，任务失败自动退还
- 静态 API Key 不扣费，仅动态用户 Key 计入

## 7. 状态流转

```
queued → processing → completed
                   ↘ failed
```

| 状态 | 含义 | 说明 |
| :-- | :-- | :-- |
| `queued` | 已入队 | 任务已创建，等待处理 |
| `processing` | 生成中 | 视频正在生成，`progress` 字段反映进度 |
| `completed` | 已完成 | 视频生成成功，`video_url` 可用 |
| `failed` | 已失败 | 生成失败，`error` 字段包含原因 |

---

## 8. 注意事项

1. **单图与多图字段不可混用**
   - JSON 格式：单图用 `image_reference`，多图用 `image_references`，不能同时出现
   - Multipart 格式：单图用 `input_reference`，多图用 `input_references`，不能同时出现
   - JSON 与 Multipart 字段名不通用

2. **image_url 支持 HTTP URL 和 data URI**
   - HTTP/HTTPS 链接：`https://example.com/image.jpg`
   - Base64 data URI：`data:image/jpeg;base64,/9j/4AAQ...`
   - 同一请求中可混合使用两种格式（如多图场景中一张用 URL、一张用 base64）
   - URL 方式对服务端更友好（不占内存），base64 适合图片不对外暴露的场景

3. **推荐轮询间隔 2-5 秒**
   - 创建任务后建议每 2-5 秒轮询一次 `GET /v1/videos/{video_id}`
   - 状态为 `completed` 或 `failed` 时停止轮询
   - 建议设置最大等待时间（如 5-10 分钟），超时后放弃

4. **时长参数合法值**
   - `seconds` 仅接受 `6`、`10`、`12`、`16`、`20` 五个值
   - 不合法的值会返回 400 错误

5. **错误返回格式**
   - 统一使用 OpenAI 风格错误体：
     ```json
     {
       "error": {
         "message": "seconds must be one of [6, 10, 12, 16, 20]",
         "type": "invalid_request_error",
         "param": "seconds",
         "code": null
       }
     }
     ```

6. **SDK 接入**
   - OpenAI 官方 SDK 无 `/v1/videos` 对应方法，需自行封装 HTTP 调用
   - 设置 `base_url` 指向本服务即可
